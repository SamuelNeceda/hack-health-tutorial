# A01:2021 - Broken Access Control

Broken Access Control (BAC) represents the most serious risk based on the OWASP Top 10:2021
security risk of web applications. As a source of vulnerabilities of this type, the
was found in up to **94.5%** of the tested applications.

## Vulnerability definition
Access Control is a way how web applications grant rights and access to functionality to different users.
Unauthorized access or change it to administrator access can arise from:
* inadequate authentication of credentials
* misconfiguration of sensitive data
* uncontrollable redirects in web application
* unauthorised access to cookies

## Vulnerability prevention
* avoid hard coding of rights or access
* implement adequate validation of user input
* restrict the readability of directories that are accessible from the URL
